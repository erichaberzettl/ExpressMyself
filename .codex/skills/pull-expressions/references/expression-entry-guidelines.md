# Expression Entry Guidelines

## Field guidance

- `id`: Stable, unique, ASCII-safe where practical. Use the language code, a slugified expression, and an index when collisions are possible.
- `language`: Use the app language code. If the app does not support it yet, document the needed language-code addition separately.
- `expression`: Preserve the expression as users would see it, including accents and required particles.
- `literalTranslation`: Use when the literal image is useful. Omit if it would confuse more than help.
- `meaning`: Explain the idiomatic or pragmatic meaning in clear English. Avoid circular definitions.
- `usageNote`: Add the real insight: register, tone, context, grammar pattern, region, caution, or common scenario.
- `exampleSentence`: Use natural target-language phrasing. It may be source-derived, lightly adapted, or newly composed only when the meaning is well-supported.
- `exampleTranslation`: Translate the example's intent into natural English.
- `difficulty`: Use `basic` for common, transparent, or broadly useful expressions; use `intermediate` for figurative, register-sensitive, regional, or less transparent expressions.
- `tags`: Include content and source tags. Recommended tags include `idiom`, `colloquial`, `proverb`, `informal`, `regional`, `slang`, `imported`, `reviewed`, and a source slug.
- `contentType`: Use `idiom` for figurative or fixed multiword expressions, `colloquialism` for informal everyday language, and `word` for single-word entries with cultural or pragmatic value.

## Red flags

Reject or quarantine entries with:

- Empty or punctuation-only meanings.
- Definitions copied from source pages without license clarity.
- Examples that merely say "People use X in conversation."
- Source expressions that are page headings, category names, titles, or navigation text.
- Literal translations presented as meanings.
- Unmarked offensive, archaic, vulgar, or region-specific usage.
- Meaning, example, and translation that contradict each other.
- Duplicate expressions unless the meanings or regions are genuinely distinct.

## Review prompts

Use these checks during manual review:

- Would a learner know when to say this after reading the entry?
- Is the usage note different from the meaning?
- Is the example something a native or fluent speaker might actually say?
- Does the translation teach the pragmatic meaning?
- Is any cultural, regional, or politeness risk visible?
